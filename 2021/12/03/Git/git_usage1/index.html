I. 常用命令

  • ^ 表示上一个提交

  • ~n 表示上 n 个提交

  • git reset 用于本地

  • git revert 用于远程仓库
  • git cherry-pick <提交号>… 基于 head 所在的位置

  • 交互式 rebase 指的是使用带参数 --interactive 的 rebase 命令, 简写为 -i

  • Git branch -f three c2

    强制把 three 分支移动到c2

IV. 分支

分支合并冲突（merge conflict）

合并分支的时候，两个分支在同一个文件的同一个位置有两种不同的修改。git 无法替我们决定使用哪一个，必须人为决定。

<<<<<<<<head 和 ==== 之间的是当前分支的代码

==== 和 >>>>>>>>hot-fix 之间的是 hot-fix 分支的代码

修改的时候要把<<<<<head 等都删掉，保留自己想要的代码

VII. 分支引用

git checkout 是把 head 移动到指定位置

Head 指向的是当前所在的最后一次commit

可以通过 git checkout pushed 把 head 切换到 pushed 这里，就变成 pushed*，* 表示 head 当前在这里
